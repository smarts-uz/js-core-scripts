# Changelog

All notable changes to this project are documented here. The format is based on
[Keep a Changelog](https://keepachangelog.com/), and the project follows
[Conventional Commits](https://www.conventionalcommits.org/) — so this file can
be regenerated from the git history.

## [Unreleased]

### Added

- **Per-method runner architecture** — every public static method of every
  `utils/` class is exposed as a standalone runner at `runs/<Class>/<method>.mjs`,
  generated by `runs/_generate.mjs`. Replaces the per-operation root scripts.
- **Contract + OLX runners** — the two former `cmd/` sub-projects were ported into
  self-contained runners that call `utils/` directly: `runs/Olx/*` (OLX scrape
  pipeline over `Chromes`/`Puppe`/`Phone`) and `runs/Word/contract.mjs`,
  `runs/Yamls/contract{Fill,Update}.mjs`, `runs/Excels/contract{,Convert}.mjs`
  (single `--yaml` or batch `--all`). Covered by `tests/ported-runners.test.js`.
- Auto-generated `README.md` (`runs/_gen-readme.mjs`) — one section per class and
  method, with JSDoc, run command, and parameter tables.
- **Project quality tooling** — ESLint (flat config), Prettier (with
  format-on-save and a Husky + lint-staged pre-commit hook), `.editorconfig`,
  `.nvmrc`, and Jest coverage thresholds.
- **CI/CD** — GitHub Actions (`lint-and-format` on Ubuntu, `test` on Windows),
  Dependabot, PR/issue templates, and `CODEOWNERS`.
- Governance docs — `LICENSE` (MIT), `CONTRIBUTING.md`, this `CHANGELOG.md`.

### Changed

- Each runner is now a complete standalone script that calls
  `<Class>.<method>(...)` directly (no shared runtime engine).
- Data folders (`data`, `bank`, `cost`) are read straight from `conf/` — the code
  (`utils/didox.js` → `../conf/data/*.json`, `utils/Yamls.js` →
  `conf/bank`/`conf/cost`) and tests were updated; no root-level symlinks.
- `runs/_wire-shell.mjs` / `runs/_wire-launch.mjs` derive the project `ROOT`
  relatively (`import.meta.dirname`) instead of a hardcoded absolute path.
- `launch.json` presets keep two examples per contract/OLX command (single +
  batch / two data sources), each labelled with its firm — not one per firm.

### Fixed

- `utils/Logs.js` writes rotating logs to the OS temp dir, never leaking a `logs/`
  folder into the working directory.
- `runs/Didox/*` import the real `didox` export from `didox.js` (was a
  non-existent `Didox`).
- Debug-terminal `${env:…}` freeze: debug configs stay `integratedTerminal`; the
  terminal profile is pinned to Command Prompt (cmd.exe) in `.vscode/settings.json`
  (VS Code's PowerShell default was the cause), fixing the hang on launch.

### Removed

- `cmd/js-winax-contract/` and `cmd/js-scraper-olx.uz/` (ported into `runs/`).
- Stray `launch.json` debug configs pointing at non-existent tools
  (`ai-rename.js`, `ai-rename-gemini.js`, `chat-export.js`); the Excel
  replace/recalc/font launcher entries; and the redundant `Readme.txt` scratch file.

### Security

- Credentials migrated from `config.yml` / source into a gitignored `.env`
  (read via `utils/Secrets.js`). Tokens previously committed to history must be
  rotated — see project notes.
